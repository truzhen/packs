# 用户使用手册

说明 Owner 如何启用、配置、运行和停用 Pack。

## 1. 软件导入与静态审查

Owner 在 Pack Studio 导入作者工程后，平台会自动执行安全性与合规性审查：
1. 打开 Truzhen **Pack Studio (模块 14)** 控制台。
2. 点击 **“导入本地工程”**，选择本模板根目录。
3. 平台将自动进行 manifest、schema、DAG、permission 和 raw secret 静态审查。

## 2. 启用与配置

Pack 通过审查后，只能生成启用候选。正式启用必须经 Base Gate 和 OwnerDecision。
1. **新建业务对象**：进入业务对象工作台，点击“新建”，创建该 Pack 的实例（如“王总智能家居项目”）。此时系统将产生 `ObjectCreatedReceipt`，但不会自动开始外部操作。
2. **启动流程**：点击“启动流程”，流程引擎 (Scene Flow Engine) 将根据 DAG 图产生第一个节点的任务候选 `TaskCandidate`。

## 3. 运行流程与 Gate 终裁

- 运行时 Scene Flow Engine 根据 Business Object 状态生成 TaskCandidate、GatewayCandidate 或 ReceiptCandidate。Owner 可在候选中心查看、批准或拒绝。
- 当流程推进到 `HumanApproval` 或需要对外发送/执行的节点时，系统会自动暂停。
- 最终用户（主人）必须在 **“任务治理 (Task Governance)”** 看板中，查看自动生成的候选卡片，并点击 **“Approve (确认)”**。
- 确认通过后，对应的 Gateway 才会下发受控动作并沉淀 append-only 的回执，严禁后台越权操作。

## 4. 停用与升级

- 停用和升级也必须走候选、Gate 和 Receipt 链路。
- 正在运行的 SceneFlowRun 不会被新版本自动改变，保证历史流程的独立性。
