# 徒真（truzhen）Packs

> 让开源作者的能力，进入中国中小企业的真实工作现场。

徒真（truzhen）面向中国 3-50 人中小企业团队。我们希望把开源软件、工具链、行业经验和服务方法，做成企业买得起、用得上、信得过的工作包。

很多中小团队不缺软件清单，也不缺聊天框。他们缺的是可以直接帮团队推进客户、交付项目、处理售后、整理证据、降低风险的工作能力；同时，他们需要知道 AI 做了什么、谁确认了什么、出了问题能不能回放。

这就是 Pack 的价值：作者把自己擅长的能力做成可审核、可安装、可试用、可回放、可付费的 Pack；企业在徒真里安装后，真实动作仍要经过主人确认、受控调用和回执记录。作者负责能力，徒真负责门控、回执、市场和分发边界。

## 谁适合来做 Pack

- **开源软件作者**：你维护 CLI、SDK、MCP Server、自动化脚本、模型工具、数据处理工具或 sidecar。
- **行业高手**：你知道销售、客服、交付、法务、财税、本地服务、工程项目里一件事该怎么推进。
- **角色设计者**：你能把服务风格、判断框架、沟通方法和质询习惯沉淀成可复用角色。

如果你的项目能帮销售团队多跟进客户、帮客服团队少漏单、帮交付团队管住项目、帮法务财税团队降低风险，徒真希望把它带到真实企业现场。

## 作者能获得什么

- **真实企业场景**：Pack 面向中国中小企业的日常工作，而不是只停留在 README、Star 和 issue 里。
- **平台收益机会**：开源项目 / 行业方法 -> Pack 化 -> 运营审核与试装 -> 市场上架 -> 企业购买、订阅或买断 -> 作者获得平台收益。具体收益以市场规则、审核结果和用户购买为准。
- **交付压力降低**：你不用把项目改造成大而全 SaaS，也不用独自承担客户权限、审计、回执和企业分发。
- **边界更清楚**：Pack 只声明能力和边界，真实发送、真实执行、正式记忆和正式业务结果仍由徒真受控处理。

## 三类 Pack

| 类型 | 适合作者 | 用户买到的是什么 |
| --- | --- | --- |
| 能力包 | 工具、软件、MCP Server、CLI、SDK、sidecar 作者 | 一个可被徒真受控调用的专业能力，例如文档解析、客户资料整理、项目快照读取、内容生成、工单辅助。 |
| 角色包 | 顾问、运营负责人、客服主管、行业专家 | 一个能以特定风格和判断框架提出建议、质询和草稿的 AI 角色。 |
| 场景包 | 行业专家、交付团队、垂直软件作者 | 一个行业工作台，把流程、对象、角色、能力和证据要求组织成完整工作方式。 |

这三类已经覆盖徒真的创作模型。新想法通常不是第四类 Pack，而是某个能力、角色或场景的一部分。

## 企业为什么敢用社区作者的 Pack

徒真不是普通插件市场，也不是 prompt 货架。Pack 不能直接拿到用户系统的控制权，也不能自行把候选变成正式结果。

企业安装 Pack 后，关键动作仍要经过：

- 主人确认：高风险动作需要用户明确确认。
- 受控调用：外部软件、模型、沟通和执行由徒真网关接入。
- 回执记录：重要动作留下可回放的证据和结果。
- 运营审核：进入市场前需要审核、试装和风险说明。

这让企业敢试用陌生作者的能力，也让作者更容易把能力卖给真实客户。

## 快速开始

1. 读 [三类 Pack 怎么选](docs/pack-types.md)，判断你的资产属于能力包、角色包还是场景包。
2. 读 [作者指南](docs/author-guide.md)，确认目标客户、适用场景、风险边界和收益预期。
3. 从 `templates/scene-pack-software-template/` 或现有样板开始，写清 Pack 能做什么、适合谁、需要哪些外部能力、哪些动作必须确认。
4. 按 [提交 Pack](docs/submit-pack.md) 做本仓检查，确认没有真实凭据、运行态数据库、provider 实现或构建产物。
5. 通过 PR 或官方审核入口提交。审核口径见 [审核与上架](docs/review-and-listing.md)，收益机制见 [收益与市场](docs/revenue-and-market.md)。

## 安全边界

- Pack 只声明能力、流程、角色和边界，不拥有最终审批权。
- AI、角色包和场景包只提出候选、草稿、质询和风险提示。
- 真实发送、真实执行、正式记忆、正式任务和正式业务结果必须回到徒真受控链路。
- 未接通的外部能力只能标记 `blocked / provider_missing / not_ready`，不能假装可用。
- 订单、支付、License / Entitlement、下载分发和正式上架状态不归本仓；本仓只保存 Pack 资产和可审核材料。
