## 写给还在犹豫的你

你可能维护着一个不错的开源项目：一个 CLI、一个 MCP Server、一套自动化脚本。它解决的问题很真实，Star 也在涨——但它给你带来的，大多止步于此。

你也可能不写代码，却把一个行业里的活儿干得比谁都明白：怎么跟进客户、怎么管住项目、怎么把一单售后处理得漂亮。这些经验带出过徒弟，却从来没有变成过你的产品。

徒真（truzhen）想改写这两件事的结局。

## 我们在做什么

中国有几千万个 3-50 人的小团队：销售团队、客服团队、装修队、家政公司、代账事务所、工程项目组。他们不逛 GitHub，不研究开源项目，但他们每天都在为具体的事头疼，而且愿意为「能直接把事办了」的东西付钱。

徒真是他们的 AI 工作平台。**Pack，就是你把能力送到他们手上的方式**：把你最擅长的东西做成一个企业可以安装、试用、购买的工作包。你负责能力本身；安装、权限、审核、留痕、收款和分发，徒真来管。

## 三类 Pack，对应三种作者

| 你是谁 | 你做的 Pack | 企业买到的是 |
| --- | --- | --- |
| **开源软件作者**——手里有 CLI、SDK、MCP Server、自动化工具 | 能力包 | 一个随取随用的专业能力：解析一摞 PDF、整理客户资料、生成一条视频、读取项目快照 |
| **行业高手**——把销售、客服、交付、法务、财税的活儿干明白了 | 场景包 | 一个拿来就能开工的行业工作台：流程、角色、要留的证据，全都替他们想好了 |
| **角色设计者**——能把服务风格和判断框架讲成方法 | 角色包 | 一个有你风格的 AI 角色：像老师傅一样提建议、挑毛病、打草稿 |

就这三类，没有第四类。拿不准选哪个？[三类 Pack 怎么选](docs/pack-types.md) 两分钟讲清楚。

## 企业为什么敢装一个陌生人做的东西

这是所有插件市场都没解决好的问题，也是徒真存在的理由。

在徒真里，Pack 碰不到企业系统的方向盘。AI 和 Pack 负责出主意、打草稿；真正要紧的动作——发出去一条消息、改动一条正式数据——必须老板亲手确认才会发生。每一步做了什么，都留下可以回看的记录；上架之前，运营会逐个审核、真机试装。

所以企业敢试你的 Pack，不是因为认识你，而是因为平台的规矩替你作了保。**信任不需要你自己挣，规则替你背书。**

## 你能得到什么

- **一个你自己够不着的市场**：你的能力进入不逛 GitHub 的真实企业，在日常工作里被使用，而不是在 README 里被路过。
- **收益**：Pack 通过审核上架后，企业购买、订阅或买断，你获得平台分成。收益取决于审核结果和真实购买，我们不承诺固定收入——但我们承诺这条路是通的。
- **只做你擅长的事**：不用把项目改造成 SaaS，不用自建权限、审计和客服，不用一个人扛企业交付。
- **清清楚楚的边界**：Pack 只声明能力，出了问题有记录可查，责任分得清。

## 怎么开始

1. 看一眼 [三类 Pack 怎么选](docs/pack-types.md)，给你的能力找到位置。
2. 读 [作者指南](docs/author-guide.md)，想清楚：服务谁、解决什么、需要什么。
3. 从 `templates/scene-pack-software-template/` 复制一份最小结构，照着现成的包写。
4. 按 [如何提交](docs/submit-pack.md) 自查一遍，提交 PR 或走官方审核入口。
5. 等审核和试装通过，上架，开始被真实的团队使用。

以上均为样板，用来学习结构和口径；完整清单见 [MODULES.md](MODULES.md)。

## 我们的约定

三条底线，保护企业，也保护你：

1. **Pack 只声明能力**，不夹带执行代码、密钥或用户数据。
2. **没接通的能力诚实标注**，不假装可用；宁可少说，不可说谎。
3. **要紧的动作留下记录**，企业能回放核对，你也说得清自己没越界。

完整的技术边界和口径见 [作者指南](docs/author-guide.md) 与 [AGENTS.md](AGENTS.md)。
