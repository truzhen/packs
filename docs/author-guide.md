# 徒真（truzhen）Pack 作者指南

徒真（truzhen）Pack 面向中国 3-50 人中小企业团队。适合被 Pack 化的资产通常不是“又一个聊天提示词”，而是能在真实工作现场解决问题、能讲清风险、能留下证据的专业能力。

## 谁适合成为作者

- 开源软件作者：你维护 CLI、SDK、MCP Server、自动化脚本、数据处理工具、模型工具或 sidecar。
- 行业专家：你知道销售、客服、交付、法务、财税、项目管理、本地服务等场景里一件事应该怎么推进。
- 角色设计者：你能把服务风格、判断框架、沟通方法和质询习惯沉淀成可复用角色。

## 一个好 Pack 应该说清什么

1. 服务谁：目标行业、团队规模、典型岗位和使用场景。
2. 解决什么：用户不用懂底层软件，也能看懂这个 Pack 能让哪件事变快、变稳或变可追责。
3. 需要什么：Provider、外部软件、账号、权限、知识来源和人工核验要求。
4. 不能做什么：哪些动作必须 `blocked / provider_missing / not_ready`，哪些动作永远不能自动执行。
5. 怎么留痕：哪些候选、Gate、Gateway 调用、Evidence 和 Receipt 能让企业复核发生了什么。

## 收益预期

作者可以通过徒真 Pack 市场获得平台收益，但这不是固定收入承诺。收益取决于 Pack 是否通过运营审核、是否能被试装验证、是否满足真实客户场景、市场规则和企业购买情况。

更好的起点是先交付一个小而强的 Pack：目标客户清楚、风险边界清楚、能跑出受控阻断或回执证据，然后再扩展更多行业流程。

## 最小可提交形态

- 能力包：能力说明、ProviderRequirement、风险等级、输入输出、门控和回执口径。
- 角色包：角色用途、口吻、判断框架、知识引用、禁区和候选输出范围。
- 场景包：manifest、flow、role slots、role packs、capabilities、必要知识索引和 README。

只写愿景、营销文案、mock-only 演示或候选字段，不能算可上架 Pack。
