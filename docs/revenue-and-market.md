# 收益与市场

你的开源项目值得的不只是 Star。徒真（truzhen）想让作者的能力进入真实企业的日常工作，并从中获得可持续的收益。先把话说实：这里的「收益」不是固定收入承诺，而是一个受审核、上架、购买和分账规则约束的平台机制——路是通的，但要一步步走。

## 收益路径

```text
开源项目 / 行业方法
-> Pack 化
-> 运营审核与试装
-> 市场上架
-> 企业购买、订阅或买断
-> 作者获得平台收益
```

## 为什么企业会买

中国中小企业团队通常不缺软件列表，缺的是能直接落进工作现场的能力：能帮人推进具体任务、能控制风险、能留下证据、出问题能复盘。徒真的价值是把陌生作者的 Pack 放进受控调用和回执体系中，让企业敢试、敢买、敢持续使用。

## 作者需要承担什么

- 交付真实有用的 Pack 声明、说明和验收材料。
- 诚实标注 Provider 依赖、风险和不可用状态。
- 不承诺绕过审核、绕过 Owner 确认或直接操作用户系统。
- 保持上架材料、版本说明和支持入口可理解。

## 平台承接什么

- Pack 审核、试装和上架流程。
- 市场展示、购买、订阅或买断机制。
- 让企业放心的那套机制：关键动作过老板确认、留可回放的记录、安装 / 启用 / 停用有清楚边界。
- 订单、支付、授权（License / Entitlement）和下载分发，由徒真官方云端承载。

## 诚实边界

作者可以获得平台收益，但不能写确定性收入、跳过审核上架或平台兜底购买这类承诺。具体收益以市场规则、运营审核、用户购买、退款和分账结算为准。
