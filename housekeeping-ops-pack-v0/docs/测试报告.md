## 二、运行的生命周期（实跑链路）

```
开始
 → 受理客户保洁咨询          [business_object] → BusinessObjectWriteCandidate
 → 家政顾问出服务方案建议      [advice / slot=housekeeping_consultant] → AdviceCandidate
 → 质检以挑剔视角质询          [challenge / slot=quality_auditor]      → AdviceCandidate
 → 排期与报价能力出价          [capability] → CapabilityInvocationCandidate

 → Owner 审批                 [human_approval] → 等待 → 真 01 Gate → FormalTask
 → 向客户发送服务确认          [communication] → CommunicationDraftCandidate(受控)
 → 派工上门执行保洁            [execution]     → ExecutionIntentCandidate(受控)
```

## 四、发现与处置

### 🔴 Bug-1（已修）：标准存图纸路径丢弃 `slot_ref`，多角色场景荚无法保存

- 现象：`POST /v3/scene-flow/specs` 保存含 advice/challenge 协作节点的图纸时，校验器 Rule 16 报
  `COLLAB_NODE_MISSING_SLOT_REF`，即便图纸里写了 `slot_ref`。
- 根因：`backend/internal/sceneflow/spec/converter.go` 的 `MixedNode`（前端 canvas 混合包解析结构）
  **没有 `SlotRef` 字段**，`knownFields` 也未列 `slot_ref`。`SplitPackage` 把 `slot_ref` 丢进 `ExtraFields`，
  `node.SlotRef` 恒为空 → 校验器判定协作节点缺 slot_ref 必拒；即便绕过校验，runtime 也无法解析角色。
- 影响面：这是**所有多角色场景荚的通用阻断**——王总报价 Demo 因无 advice 节点未触发，本家政包是首个踩到的真实多角色包。
  Pack Studio canvas 保存走的也是这条 `SplitPackage` 路径，受同一 Bug 影响。
- 修复：`converter.go` 给 `MixedNode` 补 `SlotRef` + `KnowledgeAsOfDate` 字段；`SplitPackage` 拷贝两字段到
  `SceneFlowNode`；`knownFields` 加 `slot_ref` / `knowledge_as_of_date`；`MergePackage`（反向回前端）同步补回，避免往返丢字段。
- 回归：`./backend/tests/devserver`、`./backend/tests/sceneflow`、`./backend/tests/packstudio` 全绿；本包 E2E 由此从红转绿。

### 🟢 Round-2 完善点-2（已闭环）：角色包真实上架 + 槽位绑定 + 本地模型 → advice `live=true`

- 第一轮现象：两个角色节点产 AdviceCandidate（带 slot_ref），但 `blocked_reason=slot_binding_missing_or_expired`（角色包未上架/未绑定）。

## 五、本包（Pack）自身的 round-2 完善建议

1. **补对照确认门（compare_gate）**：顾问建议 vs 质检质询应进一个 `compare_gate` 节点产 pending-owner GateCandidate，
   让 Owner 对照两个角色的输出裁定，而非两条 advice 各自独立。当前流程是顺序 advice→challenge，缺显式对照门。
2. **补条件分支**：`owner_approval` 现仅 `approved` 一条出边；应补 `rejected`/`revise` 分支回到报价或派工重整，
   覆盖「Owner 打回重报」的真实客服路径。
3. **补客户对象字段与 ReadModel 胶囊**：`intake_inquiry` 应落结构化客户服务对象（房型/面积/品类/预算/联系方式 ref），
   前端胶囊页投影服务进度时间线。
4. **报价/排期能力接真 provider**：`truzhen.capability.quote` / `scheduling` 应接 04 能力中心真实 invoke，
   产带价格/时间窗的结构化候选，而非节点标题占位。
5. **回执→记忆候选**：服务完成回执应可转记忆候选（客户偏好/复购线索），Owner 确认后入 Memory（事务主线收口）。
